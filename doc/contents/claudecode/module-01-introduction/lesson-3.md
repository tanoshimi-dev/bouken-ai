# 基本操作ガイド

## はじめに

このレッスンでは、Claude Codeの基本的な操作方法を学びます。ファイルの読み込み、編集、新規作成、そしてコマンドの実行まで、日常的な開発作業で使う操作を実践的に解説します。

## ファイルの読み込み

Claude Codeはプロジェクト内のファイルを自動的に認識し、読み込むことができます。

### 特定のファイルを読む

```bash
# Claude Codeに特定のファイルの内容を確認してもらう
> src/app.tsのコードを見せて

# ファイルの特定の部分に注目してもらう
> package.jsonの依存関係を確認して

# 複数ファイルの関連を分析
> src/components/以下のコンポーネントの構成を説明して
```

### プロジェクト構造の把握

```bash
# プロジェクト全体の構造を理解してもらう
> このプロジェクトのディレクトリ構造を説明して

# 特定のパターンのファイルを探す
> テストファイルの一覧を見せて
```

## ファイルの編集

Claude Codeの最も強力な機能の一つが、ファイルの直接編集です。

### 既存ファイルの修正

```bash
# バグの修正を依頼
> src/utils/calculate.tsの計算ロジックにバグがあります。
> 小数点の丸め処理が正しくないので修正して。

# コードの改善を依頼
> src/api/handler.tsのエラーハンドリングを改善して。
> try-catchブロックで適切なエラーメッセージを返すようにして。
```

Claude Codeはファイルを編集する前に、変更内容を表示して確認を求めます。承認すると変更が適用されます。

### 複数ファイルの一括編集

```bash
# 関連する複数ファイルの変更
> すべてのコンポーネントファイルにPropTypesの型定義を追加して

# リファクタリング
> utilsフォルダの関数をすべてTypeScriptの型付きに変換して
```

## ファイルの新規作成

新しいファイルやコンポーネントの作成も自然言語で指示できます。

### 新しいコンポーネントの作成

```bash
# Reactコンポーネントの作成
> src/components/UserProfile.tsxを作成して。
> ユーザー名、メールアドレス、アバター画像を表示するコンポーネントにして。

# テストファイルの作成
> UserProfile.tsxのユニットテストを作成して。
> src/components/__tests__/UserProfile.test.tsxに配置して。
```

### 設定ファイルの作成

```bash
# ESLintの設定ファイルを作成
> このプロジェクトに適したESLint設定ファイルを作成して。
> TypeScriptとReactのルールを含めて。

# Docker関連ファイルの作成
> このNode.jsプロジェクト用のDockerfileとdocker-compose.ymlを作成して。
```

## コマンドの実行

Claude Codeを通じてシェルコマンドを実行できます。

### ビルドとテスト

```bash
# テストの実行
> テストを実行して結果を見せて

# ビルドの実行
> プロジェクトをビルドして、エラーがあれば修正して

# リンターの実行
> ESLintを実行してエラーを確認して
```

### Git操作

```bash
# ステータスの確認
> gitの状態を確認して

# 変更のコミット
> 今の変更を適切なコミットメッセージでコミットして

# ブランチの作成
> feature/user-authenticationという名前で新しいブランチを作成して
```

### パッケージ管理

```bash
# パッケージのインストール
> axiosとreact-queryをインストールして

# 依存関係の確認
> 未使用の依存関係がないか確認して
```

## 操作のコツ

### 具体的に指示する

曖昧な指示よりも、具体的な指示の方が良い結果が得られます。

```bash
# 曖昧な指示（あまり良くない）
> コードを改善して

# 具体的な指示（良い例）
> src/api/users.tsのfetchUsers関数にエラーハンドリングを追加して。
> ネットワークエラーと認証エラーを区別して処理するようにして。
```

### コンテキストを提供する

```bash
# コンテキスト付きの指示
> 現在、ユーザー登録機能を実装しています。
> バリデーションロジックをsrc/validators/user.tsに作成して。
> メールアドレスの形式チェックとパスワードの強度チェックを含めて。
```

### 段階的に進める

大きなタスクは段階的に分けて指示すると効果的です。

```bash
# ステップ1
> まず、データベーススキーマを設計して

# ステップ2
> 次に、そのスキーマに基づいたモデルクラスを作成して

# ステップ3
> 最後に、CRUD操作のAPIエンドポイントを実装して
```

## まとめ

このレッスンでは、Claude Codeの基本操作として、ファイルの読み込み、編集、新規作成、コマンド実行の方法を学びました。これらの基本操作を組み合わせることで、日常的な開発作業を効率的に進めることができます。次のモジュールでは、より効果的にClaude Codeを使うためのプロンプトエンジニアリングについて学びます。
