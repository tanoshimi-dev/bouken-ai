# 効果的なプロンプトの書き方

## はじめに

Claude Codeの能力を最大限に引き出すためには、適切なプロンプト（指示文）を書くことが重要です。このレッスンでは、効果的なプロンプトの構造、具体性の重要性、そしてコンテキストの提供方法について学びます。

## プロンプトの基本構造

良いプロンプトは、以下の要素で構成されます。

1. **目的**: 何を達成したいのか
2. **コンテキスト**: 背景情報や制約条件
3. **具体的な要件**: 期待する結果の詳細
4. **出力形式**: どのような形で結果が欲しいか

### 基本構造の例

```bash
# 悪い例（曖昧すぎる）
> 関数を作って

# 良い例（構造化されている）
> ユーザー入力のバリデーション関数を作成してください。
>
> 要件:
> - メールアドレスの形式チェック
> - パスワードは8文字以上、大文字小文字数字を含む
> - ユーザー名は3-20文字の英数字のみ
>
> TypeScriptで記述し、src/validators/userValidation.tsに配置してください。
```

## 具体性の重要性

プロンプトが具体的であるほど、期待通りの結果が得られます。

### レベル別の具体性

```bash
# レベル1: 抽象的（結果が予測しにくい）
> APIを作って

# レベル2: やや具体的
> ユーザー管理のREST APIを作って

# レベル3: 具体的（推奨）
> Express.jsを使ったユーザー管理REST APIを作成してください。
> エンドポイント:
> - GET /api/users - ユーザー一覧取得
> - GET /api/users/:id - ユーザー詳細取得
> - POST /api/users - ユーザー作成
> - PUT /api/users/:id - ユーザー更新
> - DELETE /api/users/:id - ユーザー削除
> TypeScriptで記述し、Zodでバリデーションを行ってください。
```

### 技術的な要件を明示する

```bash
# 技術スタックを指定
> 以下の技術スタックでログイン機能を実装してください:
> - フレームワーク: Next.js 14 (App Router)
> - 認証: NextAuth.js v5
> - データベース: Prisma + PostgreSQL
> - スタイリング: Tailwind CSS
>
> ソーシャルログイン（Google, GitHub）に対応してください。
```

## コンテキストの提供

Claude Codeに十分なコンテキスト（文脈情報）を与えることで、プロジェクトに適した回答が得られます。

### 既存コードへの参照

```bash
# 既存のパターンに合わせてもらう
> src/api/products.tsを参考にして、同じパターンで
> src/api/orders.tsを作成してください。
> エラーハンドリングとログ出力のパターンも揃えてください。
```

### 制約条件の明示

```bash
# 制約条件を伝える
> このプロジェクトでは以下のルールに従ってください:
> - 外部ライブラリの追加は最小限にする
> - 関数は1つの責務のみを持つ（単一責任原則）
> - すべての公開関数にJSDocコメントを付ける
> - エラーメッセージは日本語で記述する
```

## 効果的なプロンプトパターン

### パターン1: 「〜として」の役割指定

```bash
> シニアTypeScriptエンジニアとして、このコードをレビューしてください。
> パフォーマンス、セキュリティ、保守性の観点からフィードバックをお願いします。
```

### パターン2: 「まず〜、次に〜」のステップ指定

```bash
> まず現在のコードの問題点を分析してください。
> 次に、改善案を3つ提案してください。
> 最後に、最も効果的な改善案を実装してください。
```

### パターン3: 「〜のように」の例示

```bash
> 以下のような形式でAPIレスポンスを返すようにしてください:
> {
>   "status": "success",
>   "data": { ... },
>   "meta": { "total": 100, "page": 1, "limit": 20 }
> }
```

### パターン4: 否定条件の活用

```bash
> ユーザー検索機能を実装してください。
> ただし、以下は避けてください:
> - SQLインジェクションの脆弱性
> - N+1クエリ問題
> - 暗号化されていないパスワードの取得
```

## よくある失敗と改善方法

### 失敗1: 情報が不足している

```bash
# 失敗例
> エラーを直して

# 改善例
> src/api/auth.tsの42行目でTypeErrorが発生しています。
> エラーメッセージ: "Cannot read properties of undefined (reading 'email')"
> ユーザーオブジェクトがnullの場合に発生するようです。修正してください。
```

### 失敗2: 一度に多くを求めすぎる

```bash
# 失敗例（一度に全部）
> フルスタックのECサイトを作って

# 改善例（段階的に）
> まず、商品一覧ページのコンポーネントを作成してください。
> 商品カード、フィルタリング、ページネーションを含めてください。
```

## まとめ

効果的なプロンプトを書くための鍵は、「具体性」「構造化」「コンテキスト」の3つです。曖昧な指示を避け、目的・要件・制約条件を明確に伝えることで、Claude Codeから質の高い結果を得ることができます。次のレッスンでは、コンテキスト管理のテクニックについて深く学びます。
