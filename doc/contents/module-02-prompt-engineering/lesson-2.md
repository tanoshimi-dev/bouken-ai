# コンテキスト管理

## はじめに

Claude Codeを効果的に使うためには、適切なコンテキスト（文脈情報）を管理することが重要です。このレッスンでは、CLAUDE.mdファイルの活用方法、プロジェクトコンテキストの設計、そして会話の管理テクニックについて学びます。

## CLAUDE.mdファイルの活用

CLAUDE.mdは、Claude Codeにプロジェクトの情報を伝えるための特別なファイルです。プロジェクトのルートディレクトリに配置すると、Claude Codeが自動的に読み込みます。

### 基本的なCLAUDE.mdの構造

```markdown
# プロジェクト概要

このプロジェクトは、社内のタスク管理システムです。

## 技術スタック

- フロントエンド: React 18 + TypeScript
- バックエンド: Node.js + Express
- データベース: PostgreSQL + Prisma
- テスト: Jest + React Testing Library
- CI/CD: GitHub Actions

## ディレクトリ構造

src/
  components/   # Reactコンポーネント
  pages/        # ページコンポーネント
  api/          # APIクライアント
  hooks/        # カスタムフック
  utils/        # ユーティリティ関数
  types/        # TypeScript型定義

## コーディング規約

- 変数名・関数名はキャメルケースを使用
- コンポーネント名はパスカルケースを使用
- インデントはスペース2つ
- セミコロンは省略しない
- 文字列はシングルクォートを使用

## 重要な注意事項

- 環境変数は.envファイルで管理（.gitignoreに含まれている）
- APIのベースURLはAXIOS_BASE_URL環境変数で設定
- 日本語ロケールを前提とした日時処理を行う
```

### CLAUDE.mdの配置場所

CLAUDE.mdは複数の場所に配置できます。

```bash
# プロジェクトルート（最も一般的）
/project-root/CLAUDE.md

# ユーザーのホームディレクトリ（グローバル設定）
~/.claude/CLAUDE.md

# サブディレクトリ（特定の領域に関する情報）
/project-root/src/api/CLAUDE.md
```

優先順位は、より具体的なパス（サブディレクトリ）が優先されます。

### 効果的なCLAUDE.mdの書き方

```markdown
# プロジェクト固有のルール

## やるべきこと
- すべての関数にJSDocコメントを付ける
- 新しいコンポーネントには対応するテストファイルを作成する
- APIエラーは共通のErrorHandlerクラスで処理する

## やってはいけないこと
- anyの使用は禁止（unknown + 型ガードを使用すること）
- console.logをプロダクションコードに残さない
- インラインスタイルの使用は避ける

## よく使うコマンド
- `npm run dev` - 開発サーバー起動
- `npm run test` - テスト実行
- `npm run build` - プロダクションビルド
- `npm run lint` - リンター実行
```

## プロジェクトコンテキストの設計

### レイヤー別のコンテキスト管理

大規模プロジェクトでは、各レイヤーに応じたCLAUDE.mdを配置することで、より精密なコンテキストを提供できます。

```bash
# フロントエンド固有の指示
# src/components/CLAUDE.md
```

```markdown
# コンポーネント開発ガイドライン

## コンポーネントの命名規則
- 機能を表す名前を付ける（例: UserProfileCard）
- 汎用コンポーネントはUI/プレフィックスを付ける（例: UIButton）

## Props の定義
- すべてのPropsにinterfaceを定義する
- オプショナルなPropsにはデフォルト値を設定する

## 使用例
interface UserCardProps {
  userId: string;
  showAvatar?: boolean; // デフォルト: true
}
```

### 動的なコンテキスト提供

会話の中でコンテキストを追加することも効果的です。

```bash
# 現在の作業内容を伝える
> 現在、ユーザー認証機能のリファクタリングを行っています。
> JWT トークンの管理をCookieベースからhttpOnly Cookieに
> 変更する作業です。関連ファイルを確認してから修正を始めてください。

# エラーの文脈を伝える
> 本番環境で以下のエラーが発生しています。
> 開発環境では再現しません。
> 環境差異を考慮して原因を分析してください。
```

## 会話の管理テクニック

### 会話のリセット

長い会話を続けると、コンテキストが肥大化して精度が落ちる場合があります。

```bash
# 新しいトピックに移る前にリセット
# Claude Codeで /clear コマンドを使用

# または新しいセッションを開始
# Ctrl+C で終了し、再度 claude を起動
```

### コンテキストウィンドウの効率的な使い方

```bash
# 不要な情報を含めない
# 悪い例: ログ全体を貼り付ける
> このエラーログ全体を分析して（数千行のログ）

# 良い例: 関連部分だけを抽出
> 以下のエラーメッセージが繰り返し発生しています:
> [ERROR] 2024-01-15 10:23:45 - Connection timeout at db.connect()
> 直前のログには "Pool size exceeded" の警告があります。
```

### タスクの分割管理

```bash
# 大きなタスクをサブタスクに分割
> 認証機能の実装を以下のステップで進めたいです:
>
> ステップ1: ユーザーモデルとDBスキーマの作成
> ステップ2: 登録・ログインAPIの実装
> ステップ3: JWTミドルウェアの作成
> ステップ4: 保護されたルートの設定
>
> まずステップ1から始めてください。
```

## コンテキスト管理のベストプラクティス

### 1. 段階的な情報開示

最初から全ての情報を与えるのではなく、必要に応じて段階的に情報を提供します。

```bash
# 最初の指示
> ユーザープロフィール編集ページを作成してください。

# 追加の情報
> 画像アップロードにはS3を使用しています。
> 既存のuploadService（src/services/upload.ts）を利用してください。
```

### 2. フィードバックループ

結果を確認し、フィードバックを与えることで精度を向上させます。

```bash
# 結果を確認してフィードバック
> この実装は良いですが、以下の点を修正してください:
> 1. エラーメッセージを日本語にする
> 2. ローディング状態の処理を追加する
> 3. 入力値のサニタイズ処理を加える
```

## まとめ

コンテキスト管理は、Claude Codeを効果的に使うための核心的なスキルです。CLAUDE.mdを活用してプロジェクト情報を体系的に提供し、会話の中でも適切にコンテキストを管理することで、一貫性のある高品質な結果を得ることができます。次のレッスンでは、さらに高度なプロンプトテクニックについて学びます。
