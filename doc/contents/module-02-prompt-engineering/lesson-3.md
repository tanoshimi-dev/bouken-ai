# 高度なプロンプトテクニック

## はじめに

このレッスンでは、Claude Codeでより高度な結果を得るためのプロンプトテクニックを学びます。思考の連鎖（Chain of Thought）、ステップバイステップアプローチ、役割定義、そして反復的な改善手法を習得しましょう。

## 思考の連鎖（Chain of Thought）

Claude Codeに段階的に考えさせることで、複雑な問題をより正確に解決できます。

### 分析してから実行させる

```bash
# 思考の連鎖を促すプロンプト
> 以下のパフォーマンス問題を解決してください。
>
> まず、src/api/users.tsのfetchAllUsers関数を分析して、
> パフォーマンスのボトルネックになっている箇所を特定してください。
>
> 次に、それぞれのボトルネックについて改善案を考えてください。
>
> 最後に、最も効果的な改善案を実装してください。
```

### 推論過程を明示させる

```bash
# 理由を説明させる
> このエラーの原因を推論してください:
> TypeError: Cannot read properties of null (reading 'map')
>
> 考えられる原因を3つ挙げ、それぞれの可能性を評価した上で、
> 最も可能性の高い原因に対する修正を実装してください。
> 修正する際は、なぜその修正が適切なのかも説明してください。
```

## ステップバイステップアプローチ

複雑なタスクを明確なステップに分解して指示します。

### 機能実装のステップ分解

```bash
> ショッピングカート機能を以下のステップで実装してください:
>
> Step 1: 型定義
> - CartItem, Cart のインターフェースを定義
> - src/types/cart.tsに配置
>
> Step 2: 状態管理
> - Zustandを使ったカートストアを作成
> - 追加、削除、数量変更、クリアのアクションを実装
> - src/stores/cartStore.tsに配置
>
> Step 3: コンポーネント
> - CartItemCard: 個別商品の表示・操作
> - CartSummary: 合計金額の表示
> - CartPage: カートページ全体
> - src/components/cart/以下に配置
>
> Step 4: テスト
> - カートストアのユニットテスト
> - 各コンポーネントのレンダリングテスト
>
> 各ステップが完了したら確認してから次に進みます。
```

### リファクタリングのステップ分解

```bash
> レガシーコードのリファクタリングを安全に行いたいです:
>
> 1. まず、現在のコードの動作を確認するテストを書いてください
> 2. テストが通ることを確認してください
> 3. テストを変えずにコードをリファクタリングしてください
> 4. リファクタリング後もテストが通ることを確認してください
> 5. 変更点のサマリーを作成してください
```

## 役割定義テクニック

Claude Codeに特定の役割を与えることで、その視点からの回答を得られます。

### 専門家の役割を指定

```bash
# セキュリティ専門家として
> セキュリティエンジニアの視点で、src/api/auth.tsのコードを
> レビューしてください。以下の観点でチェックしてください:
> - SQLインジェクション
> - XSS攻撃
> - CSRF対策
> - 認証・認可の実装
> - 機密情報の取り扱い
```

```bash
# パフォーマンスエンジニアとして
> パフォーマンスエンジニアとして、このReactアプリケーションの
> レンダリングパフォーマンスを分析してください。
> 不要な再レンダリングが発生している箇所を特定し、
> React.memo、useMemo、useCallbackの適切な使い方を提案してください。
```

### コードレビュアーの役割

```bash
> 経験豊富なシニアエンジニアとして、このPull Requestの変更を
> レビューしてください。
>
> レビューの基準:
> - コードの可読性
> - エッジケースの処理
> - エラーハンドリング
> - テストの網羅性
> - 設計パターンの適切さ
>
> 問題点には重要度（高・中・低）を付けてください。
```

## 反復的改善テクニック

一度で完璧な結果を求めるのではなく、段階的に改善していくアプローチです。

### フィードバックループ

```bash
# 初回の生成
> ユーザー登録フォームのコンポーネントを作成してください。

# フィードバック1: 機能追加
> パスワード強度メーターを追加してください。
> 弱い（赤）、普通（黄）、強い（緑）の3段階で表示してください。

# フィードバック2: UX改善
> バリデーションエラーはフィールドの下にリアルタイムで
> 表示するようにしてください。サブミット時だけでなく、
> フォーカスが外れたタイミングでもチェックしてください。

# フィードバック3: アクセシビリティ
> WAI-ARIAに準拠したアクセシビリティ対応を追加してください。
> スクリーンリーダーでエラーメッセージが読み上げられるようにしてください。
```

### 比較検討アプローチ

```bash
# 複数の選択肢を比較する
> データフェッチの実装方法を3つ提案してください:
> 1. useEffect + fetch
> 2. React Query (TanStack Query)
> 3. SWR
>
> それぞれのメリット・デメリットを比較表で示してから、
> このプロジェクトに最適なものを選んで実装してください。
> 選択の理由も説明してください。
```

## 高度な組み合わせテクニック

### テンプレートパターン

繰り返し使うプロンプトをテンプレート化します。

```bash
# コンポーネント作成テンプレート
> 以下の仕様でReactコンポーネントを作成してください:
>
> コンポーネント名: [名前]
> 配置場所: src/components/[パス]
> Props:
> - [prop名]: [型] - [説明]
>
> 機能要件:
> - [要件1]
> - [要件2]
>
> 非機能要件:
> - レスポンシブ対応
> - アクセシビリティ対応
> - ユニットテスト付き
```

### 制約付き生成

```bash
# 厳密な制約を設定
> 以下の制約条件のもとで検索機能を実装してください:
>
> パフォーマンス制約:
> - 検索結果の表示は200ms以内
> - デバウンス時間は300ms
>
> コード制約:
> - 外部ライブラリの追加は不可
> - 既存のユーティリティ関数を最大限活用する
>
> UX制約:
> - 検索中はスケルトンUIを表示
> - 結果が0件の場合は提案を表示
> - キーボードナビゲーション対応
```

## まとめ

高度なプロンプトテクニックを活用することで、Claude Codeからより質の高い、より正確な結果を得ることができます。思考の連鎖で推論過程を明確にし、ステップバイステップで複雑さを管理し、役割定義で視点を制御し、反復的改善で段階的に品質を向上させましょう。これらのテクニックを組み合わせることで、Claude Codeを真のペアプログラミングパートナーとして活用できるようになります。
